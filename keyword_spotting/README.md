# Keyword Spotting with TFLite — STM32MP1

Real-time voice recognition system for the **STM32MP1-DK2** board: continuously listens to the microphone and, when it recognizes a keyword (go, stop, yes, no), logs it to a file with a precise timestamp. Can run in parallel with other programs on the same board.

---

## How it works in brief

The project is split into two parts running on different machines:

- **Training → on the PC.** The neural network is trained on the computer, which has the necessary computing power. Training downloads a ~2 GB dataset, trains a CNN for a few hours, and produces a compact file (`keyword_model.tflite`, ~430 KB).

- **Inference → on the board.** The `.tflite` file is copied to the STM32MP1, which uses it to recognize keywords in real time by listening to the microphone. The board does not need to retrain anything: it simply runs the already-trained model.

## Board setup

### Physical connection

1. Connect the **power supply** to the board
2. Connect a **USB microphone** to one of the board's USB ports
3. Connect a **USB-MicroUSB cable** between the board and the PC (the `ST-LINK` port on the board) — required for the serial connection

### Connecting to the board with PuTTY

Open **PuTTY** on the PC and configure a serial connection:
- **Connection type:** Serial
- **Serial line:** the COM port assigned to the board (e.g. `COM3` — check in Device Manager)
- **Speed:** 115200

Click **Open**. Press Enter if the terminal looks empty: you should see the login prompt. Log in as `root` (no password).

## Step-by-step

### PHASE 1 — On the PC (training)

#### 1.1 — Install dependencies
```bash
cd keyword_spotting
pip install -r requirements_pc.txt
```

#### 1.2 — (Optional) Modify the keywords
Open `config.py` and edit the `LABELS` list:
```python
LABELS = ["silence", "start", "stop", "yes", "no"]
```
- `"silence"` must always be the first element (catches everything that is not recognized)
- The others must match keywords present in the Google Speech Commands dataset
  - Available keywords: yes, no, up, down, left, right, on, off, stop, go, zero–nine, etc.
  - For custom keywords: see the "Custom Keywords" section at the bottom

#### 1.3 — Train the model
```bash
python training/train.py
```
Training automatically downloads the dataset (~2 GB).
At the end you will find `models/keyword_model.tflite` (~200–500 KB).

#### 1.4 — Evaluate the model (optional)
```bash
python training/evaluate.py
```
Displays accuracy, F1 score, and confusion matrix (`models/confusion_matrix.png`).

---

### PHASE 2 — Transfer to the MP1

#### 2.1 — Copy the project to the MP1
```bash
# from PC:
scp -r keyword_spotting/ root@<IP_MP1>:/home/root/
# or with rsync (excludes heavy training files):
rsync -av --exclude='*.keras' --exclude='*.h5' \
    keyword_spotting/ root@<IP_MP1>:/home/root/keyword_spotting/
```

#### 2.2 — Install dependencies on the MP1
```bash
# on the MP1:
cd /home/root/keyword_spotting
pip3 install -r requirements_mp1.txt
```
> **Note:** if `pyaudio` fails to compile, try:
> ```bash
> apt-get install portaudio19-dev python3-pyaudio
> ```

#### 2.3 — Test microphone and pipeline
```bash
python3 inference/test_mic.py
```
Verifies that the microphone is detected and that the MFCC pipeline works correctly.

---

### PHASE 3 — Running on the MP1

#### 3. — Manual start
```bash
cd /home/root/keyword_spotting
python3 inference/keyword_spotter.py
```

### PHASE 4 — Log analysis

#### Keyword log format
```
2026-01-26T10:32:01.412 | unix=1706262721.412 | keyword=start | conf=0.9230
2026-01-26T10:32:15.887 | unix=1706262735.887 | keyword=stop  | conf=0.9710
```
The log file is `keyword_log.txt`. To view it, run:
```bash
nano logs/keyword_log.txt
```

## Main parameters (config.py)

| Parameter | Default | Description |
|-----------|---------|-------------|
| `SAMPLE_RATE` | 16000 | Sampling rate (Hz) |
| `WINDOW_SEC` | 1.0 | Audio window duration (s) |
| `HOP_SEC` | 0.5 | Hop between consecutive windows (s) |
| `N_MFCC` | 40 | Number of MFCC coefficients |
| `THRESHOLD` | 0.85 | Minimum confidence to log |
| `LABELS` | [...] | Keywords + "silence" |

## Custom keywords

If you want to use keywords not present in the Speech Commands dataset (e.g. "record", "pause"):

1. Record ~100–200 examples per keyword using a tool like Audacity (16kHz, mono, WAV)
2. Save the files in `training/custom_data/<keyword>/clip_001.wav`, etc.
3. Integrate your data into the training by modifying `train.py` to also load this folder
4. Retrain and re-export the TFLite model

## Troubleshooting

**`No module named 'tflite_runtime'`**
→ `pip3 install tflite-runtime` or use the full `tensorflow` package on the PC.

**`ALSA lib: no soundcards found`**
→ Verify that the microphone is connected and recognized: `arecord -l`

**Poor recognition accuracy**
→ Lower `THRESHOLD` in `config.py` or retrain with more data.

**Latency too high**
→ Reduce `WINDOW_SEC` or `HOP_SEC` in `config.py` (and retrain the model).
